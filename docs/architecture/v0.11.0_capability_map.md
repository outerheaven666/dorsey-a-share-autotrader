# v0.11.0 Capability Map

本表用于说明 MVP 1-11 的能力边界。`Still simulated?` 为 `Yes` 的能力不应被理解为实盘能力。`Frozen or evolving?` 表示 v0.11.0 是否建议短期冻结。

| MVP | Capability | Main files/modules | Output artifacts | Risk controlled | Still simulated? | Frozen or evolving? | Notes |
| --- | --- | --- | --- | --- | --- | --- | --- |
| MVP 1 | 评分、组合构建、paper broker | `src/dorsey_as/scoring.py`, `factors/`, `moat/`, `valuation/`, `risk/`, `portfolio/`, `broker/paper.py` | `scores.csv`, `target_portfolio.csv`, `paper_trades.csv` | 把规则显式化，避免黑箱评分 | Yes | Evolving | 评分逻辑可继续校验，但不要扩展到真实交易。 |
| MVP 2 | 季度回测 | `src/dorsey_as/backtest/` | `backtest_equity_curve.csv`, `backtest_trades.csv`, `backtest_holdings.csv`, `backtest_metrics.csv` | 避免只看静态得分，加入组合复盘 | Yes | Evolving | 回测使用 sample data，交易现实性仍不足。 |
| MVP 3 | 数据质量与防未来函数 | `src/dorsey_as/data_quality/`, `point_in_time.py` | `data_quality_report.csv`, `point_in_time_snapshot.csv`, `backtest_audit_log.csv` | 阻止缺失、异常、未来披露数据进入流程 | Partly | Evolving | 规则可复用，但真实数据仍需重新验证。 |
| MVP 4 | 配置与 Markdown 报告 | `config/default.yaml`, `src/dorsey_as/config/`, `reporting/markdown.py` | `run_report.md`, `backtest_report.md` | 降低硬编码风险，提高解释性 | No trading | Frozen baseline | 配置结构可冻结，参数不应频繁调优。 |
| MVP 5 | HTML 报告、图表、dry-run 通知、审计 | `reporting/html.py`, `reporting/charts.py`, `notify/`, `audit.py` | `run_report.html`, `backtest_report.html`, `notify_payload.json`, `notify_summary.md`, `decision_audit_log.csv` | 提高复盘可读性和决策留痕 | Yes | Frozen baseline | 通知仅 dry-run，不是生产告警。 |
| MVP 6 | 点时数据增强、因子审计 drilldown | `factors/audit.py`, `point_in_time.py`, `scoring.py` | `factor_audit_log.csv`, `explain_*.md` | 让得分可追溯，红旗拦截可解释 | Partly | Evolving | 可作为未来真实数据接入前的审计基础。 |
| MVP 7 | Adapter Contract 测试层 | `src/dorsey_as/adapters/` | `provider_contract_report.csv`, `adapter_mapped_preview.csv`, `provider_contract_summary.md` | 防止真实 provider 直接进入主路径 | Yes | Frozen baseline | 仅 mock provider，不参与交易决策。 |
| MVP 8 | Schema Versioning 与 Contract Diff | `src/dorsey_as/schema_versioning/`, `config/schemas/` | `provider_contract_diff_report.csv`, `provider_contract_diff_summary.md` | 识别 breaking/additive/compatible schema 变化 | No trading | Frozen baseline | 只做数据接入前检查，不参与交易决策。 |
| MVP 9 | Schema Migration Metadata 与可视化 | `src/dorsey_as/schema_migration/`, `config/schema_migrations/` | `schema_migration_report.csv`, `schema_migration_summary.md`, `provider_contract_diff.html` | 解释字段迁移、deprecation 生命周期和兼容窗口 | No trading | Frozen baseline | migration metadata 不是交易信号。 |
| MVP 10 | Pre-Live Safety Gate | `src/dorsey_as/safety/` | `pre_live_safety_report.csv`, `safety_explanation.md`, `simulated_live_request_report.csv` | 默认阻断 live trading、real broker、real order、real network data | No trading | Frozen baseline | 安全闸门应保持默认阻断。 |
| MVP 11 | System Health 与 Release Checklist | `src/dorsey_as/system_health/`, `README.md` | `system_health_report.csv`, `sensitive_scan_report.csv`, `output_artifact_manifest.csv`, `release_checklist.md`, `release_notes_v0.11.0.md` | 发布前检查安全默认值、输出产物和文档边界 | No trading | Frozen baseline | 只生成本地报告，不自动发布。 |

## 可以冻结的能力

- 本地 sample CSV 运行路径。
- paper/backtest simulation 的安全边界。
- adapter contract 的 mock-only 默认模式。
- schema versioning 和 schema migration 的检查定位。
- pre-live safety gate 的默认阻断策略。
- system health 与 release checklist 的本地报告定位。

## 仍属于 sample / fixture / mock 的能力

- sample CSV scoring。
- sample historical backtest。
- mock provider contract validation。
- fixture schema diff cases。
- fixture migration cases。
- dry-run notification。

## 不参与交易决策的能力

- adapter contract validation。
- schema diff。
- schema migration metadata。
- contract diff visualization。
- system health。
- release checklist。
- sensitive scan。
- release notes draft。

这些能力用于检查、解释、发布复验和安全边界，不应被当成买卖信号。
